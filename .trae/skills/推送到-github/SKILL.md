---
name: 推送到 Github
description: 当用户需要将 代码实现推送到 GitHub 时，自动生成中文提交日志，并自动维护README.md 中的项目目录，保持项目结构清晰、可追溯、适合考研复习使用。
---

# 执行逻辑（固定流程）
1. 接收用户推送指令
2. 自动识别本次更新内容（线性表/栈/队列/树/图/排序/查找等）
3. 生成规范中文 Git 提交日志
4. 自动更新 README.md 中的「项目目录」
5. 输出可直接复制的推送命令

# 提交日志格式（必须使用中文）
格式：
[模块名] 功能描述

示例：
[线性表] 实现顺序表逆置算法（空间O(1)）
[栈] 完成括号匹配问题
[排序] 实现快速排序递归与非递归版本

# README 目录格式（固定结构）
自动维护以下结构：
# 王道408数据结构考研 — C++ 实现
## 目录
- 线性表
  - 顺序表基本操作
  - 顺序表逆置
  - 删除最小值
  - 删除所有值为x的元素
- 栈与队列
- 字符串
- 树与二叉树
- 图
- 排序
- 查找

# 推送命令（固定输出）
git add .
git commit -m "[模块] 完成XXX功能实现"
git push

# 输出风格
- 简洁、考研向
- 中文日志
- 目录清晰
- 适合个人复习仓库使用
